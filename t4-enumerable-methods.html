What is an Enumerable and how to use .group_by

7/12/2014

Let's start out with what is an Enumerable. Enumerable its a gross long word but once you understand what an enumerable is it will carry many positive connotations. An Enumerable is a simplified version of doing something like a .each do loop. Ill show you an example of one at work and then explain what is going on.
ary = [1, 2, 4, 2] ary.count{ |x| x%2==0 } #=> 3
this is an example right from the Ruby docs. What is happening here is that the method count is being called on an array. The method count without any Enumerable will just display how many items ary has. But with the provided Enumerable it is displaying how many elements in the array are divisible by 2. The |x| at the beginning is the same as in a zdo statement. It is assigning each element to that then running it through the following test and adding it to the count if the test evaluates to true.
Now for the .group_by method. The group by method is very interesting it, I'll give you an example and then explain it below. (this example comes from http://markusjais.com/ because the one on Ruby docs wasn't very good ) cats = ["Tiger", "Lion", "Puma", "Leopard", "Jaguar", "Cheetah", "Bobcat"] grouped_by_length = cats.group_by { |cat| cat.length } #{5=>["Tiger"], 4=>["Lion", "Puma"], 7=>["Leopard", "Cheetah"], 6=>["Jaguar", "Bobcat"]} What is happening here is that the array cats it getting put into a Hash that has keys of the grouping method and values that match that key. So for this example each cat is being grouped by the number of letters its name contains, or the .length of it's string. From this the group_by method says " okay we have string with 4 letters are there another strings with 4 letters? yes 1 more, lets put that into a hash called 4". It is fairly simple but could be used as a very powerful sorting method.
This is just a very simple explanation of enumerables and I don't really understand everything about them but I hope this gives you a start to understanding them.
